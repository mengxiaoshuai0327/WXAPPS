"大牛小班课"小程序主要功能：前台：零、登录系统1）用户进入小程序后，调用 wx.login 获得 openid，后端识别“是不是老用户”。2）新用户一律视为游客，游客可以查看主题、课程、授课人的详细介绍、课程表、课程评价（不许回复），点击其他任何功能都会弹出“注册会员弹窗”。3）点击 我的-注册会员 同样会弹出“注册会员弹窗”，填写昵称（用于对外展示，默认用微信昵称）、姓名、手机号（同一手机号只能注册一个账号，尽可能使用“获取手机号”能力，失败则手填+验证码），邀请码（邀请人ID号，如有）、同意用户协议、隐私条款，即可成为会员。一、课券：1）买课券，可选择买课券的数量，单张课券收费1500元。2）每个会员的课券分为：未使用、已预订、已使用、已过期、已赠送（已赠送再细分为等待领取、未使用、已预订、已使用、已过期）每个会员的我的页面显示：未使用、已预订、已使用的课券余额，下方有“点击查看全部课券信息”按钮，点击后，可以展开查看未使用、已预订、已使用、已过期、已赠送的课券使用信息和赠送明细。3）每次预订课程时课券会被占用（全天课占用两张课券，上午或下午的半天课占用一张课券），如果取消预订则占用取消。开课前三日不可取消，这个时候取消按钮线上灰色不可取消状态。4）用课券：如未在免费取消的截止时间前取消（见“二、课程”部分的 “3）取消预订“），则上课开始时间到了（上午或全天课为早上9:00，下午课为下午2：00）就会自动用掉课券（已预订转为已使用）。5）赠予课券：会员可以生成图片向他人赠予自己的课券，图片中显示“您收到大牛小班课的赠券”字样，以及赠券人ID号和课券ID号。如果被赠予人不是会员，被赠予人需先登记会员；接收时，被赠予人需要填写赠券人ID号和课券ID号。正在被占用的课券禁止赠予；课券有效期按原来的计算；即将过期（比如 3 天内）的课券赠予时要弹出风险提示；部分课券（如授课奖励、他人赠送的课券）不可赠予。6）对已使用过的课券可以申请开发票，已开过的不能再开。未使用过的课券暂不能开发票。开发票可以一张一张申请，也可以批量申请；开发票后该课券在后台维护成已开票状态，已开票的课券不能再次开票；开发票的金额是购买该课券实际支付的金额（即课券面额扣除使用优惠券的余额）。7）每个课券都要记录其起始时间和3 个月有效期结束时间，并记录其购买时所花金额（促销会导致课券实际购买价格会有差异），到期未用则只退实付金额，折扣券不退回。在到期日期的1个月前就开始每周自动发送一次提醒通知，鼓励会员尽快使用。每天统一时间触发一次过期退款。二、课程：1）课程表：每月中左右后台发布下月课程表，前台页面：最上方显示课程筛选（按照课程主题）和开课日期筛选；课程筛选下方显示具体课程：在主题下的课程标题位置显示授课人姓名，日期安排（具体样式是：2026-01-01 全天（9-17）/2026-01-01 上午（9-12）/2026-01-01 下午（14-17）的哪一种），当前报名人数/报名上限，预订按钮，详情（详情做成链接，点击后会打开后台输入的对该课程的授课人的介绍和对课程内容的介绍、课程编号）。历史课程表不可预订，不可取消，其他与可预订的课程表基本一样，比如也能点击进入详情。2）课程预订：会员点预订按钮可以预订其想上的课程，报满后不可再订，应实时更新预订人数和空位人数。点完预订后，要弹出确认框，确认后才完成预订。当用户已预订该课程时，显示【取消预订】按钮，历史课程不可取消，开课前三天内课程不可取消。3）会员取消预订：会员提前三天可免费取消（上课日前三天的00：00之前取消）。历史课程和开课前三天内的课程显示不可取消4）平台取消课程的流程：通过 系统内消息 + 短信 通知所有已预订学员；所有相关课券从已预定 → 未使用。5）主题、课程、授课人的详细介绍页面三、课后评价：1）将评价问卷做到小程序里，方便数据收集。在问卷中添加一个开放题，让学员提供点评和建议。课程结束前10分钟开启该课程的评价入口，管理员可以决定评价入口的关闭时间。在 我的-评价 中显示待评价的课程，若存在未评价的课程，则在 我的 页面中用红点提示。2）根据每次课程的课后评价的结果，为每个选择题绘制一个统计图表，图表对所有会员和游客可见3）所有点评对所有会员和游客可见，即用户点击历史课程时，可以看到课程介绍、统计图表、详细评价，每个上过课的会员对该课程的点评和建议会展示在详细评价中，只显示点评者的昵称。会员可以对某点评进行追评，做该课后评价的会员可以回复，评论和回复对所有会员和游客都可见，其他会员也可以跟评。管理员可以“隐藏/删除评论”，防止恶意评价、泄露隐私。管理员在后台可以一键使用AI总结一节课程的所有点评，经确认后可以发布在详细评价的顶部。四、邀请：1）在 我的-邀请 中有专门的邀请发出页面。2）刚上完某个课程后的会员可以在课后评价中点击邀请按钮，立即邀请朋友来体验该课程。在其他时候会员也可以随时发出邀请，可以提及一个或数个具体课程，邀请内容需涵盖课程的主题+副标题+授课人。也可邀请朋友加入本培训项目，内容就时介绍整体项目情况。不可邀请自己；任何一个会员只能有一个邀请人。3）邀请采用（1）获取管理员在后台已经维护好的邀请背景图片；（2）点击邀请后，自动在邀请图片中生成邀请人的会员id，如针对某课程发出邀请，还需要包括该课程名称，要能够区分发出邀请者是谁，方便统计接受邀请的有效返回，被邀请者来到小程序时要输入邀请人的会员id；邀请图片的背景要由管理员定期在后台上传，只能上传一张。4）个性化：发出邀请时，可以由会员输入其个人化邀请文字，替换通用的邀请文字。五、促销：1）折扣券：根据推荐表现可以发放一些促销折扣券用于抵扣需支付的课券价格，100/500两个等级。2）每个课券购买时限用一张折扣券。3）每邀请成功一人成为会员给予一张100金额的折扣券，当该会员购买课券后再给予当初邀请其加入会员的发出邀请的会员一张500金额的折扣券。六、授课人1）授课人的登录方式与普通会员一致，用户在登录小程序的时候系统要验证是否为授课人，如果是授课人，在【我的】页面还应该看到自己的排课信息和历史授课信息。2）授课人每授课一次，会免费得到一张课券，该课券不能赠予他人。3）授课人可以看到自己所授课某个同一课程编号的所有过往各期的课后评价总计后的统计。与普通会员能看到的统计结果，不同点在于普通会员仅能看到某期的统计结果。4）其他内容和功能，授课人看到的能用到的同普通会员一样。七、小程序前端底部 tab1）首页：角色相关的“任务中枢”2）课程表：专门展示月度排课、筛选主题3）排行榜4）我的八、首页1. 游客首页（1）顶部 Banner：宣传图 ，图需要使用管理员在后台维护图片，管理员可以定期更新宣传图，更新后要实时同步到小程序前端。（2）核心模块：【课程体系介绍】（16 个主题大纲），【查看课程表】按钮（3）底部固定提示：登录成为会员即可预订课程、评价课程、邀请好友等，一个【注册会员】按钮（拉起注册弹窗）。2. 会员首页（1）顶部：左边：欢迎语 + 昵称，右边：课券剩余：3 → 点击跳转“课券详情”（2）核心区域：已预定课程，滚动卡片，每个卡片显示 日期 + 星期 + 时间段、主题 + 副标题、授课人姓名、预订状态标签（已预订 / 已开课 / 待评价 / 已完成）。如果用户目前没有预定任何课程，则显示“你还没有预订课程~ 去看看课表？” +【查看课程表】按钮（3）小模块 1：待评价提醒，标题：待评价课程，内容：还有 2 节课待评价，列出最近1~2节，右侧【去评价】按钮。（4）小模块 2：课券 / 折扣券提示，如果课券余额低（比如 ≤1），显示“课券快用完了，去补充一下？”【购买课券】按钮；如果有快过期的课券/折扣券，显示“你有 1 张课券将在 3 天后到期”【去预订课程】按钮。（5）小模块 3：邀请卡片，“每成功邀请1位朋友注册会员，即可获得1张100元折扣券” “每位邀请的朋友首次购买课券，即可获得1张500元折扣券” “已累计获得200元折扣”，一个【继续邀请】按钮 → 跳“我的-邀请”。（6）小模块 4：系统消息预览，标题：系统通知，展示最新 1~2 条消息的标题，底部【查看全部消息】 → 跳到系统消息页。3. 授课人首页（1）顶部：昵称 + “授课人”标识 + 本月授课场次、平均评分，快捷入口【讲师资料】和【课程评价】（2）核心区域 A：近期授课安排，按时间排序列出未来 N 次授课，日期、时间段、主题、课程编号、报名人数 / 上限。（3）核心区域 B：评分概览，两个小卡片（课程编号 A01：本期 4.8，近 3 期平均 4.7） +【查看详情】跳转 我的-课程评价。（4）其余和会员首页一致九、排行榜1. 通用规则：- 排行榜以营销展示为主，只展示 TOP N，不展示“倒数榜”；- 数据基于课后评价和上课/邀请行为自动统计，管理员在后台手动生成、调整并发布榜单快照；- 前台可按时间维度查看榜单（如本月 / 近3个月 / 全部），始终展示最近一次已发布版本。2. 主题排行榜：- 以“主题”为单位，综合课后评价中的价值感、认知突破、应用帮助等维度，形成主题综合评分；- 仅纳入评价人数达到一定数量的主题，展示主题名称、解决的问题简介、综合评分和评价人数；- 点击可跳转到该主题的详细介绍及相关课程列表。3. 课程排行榜：- 以“课程编号”为单位，汇总同一课程编号各期课后评价，形成课程综合评分；- 展示课程编号、主题、课程副标题、授课人、综合评分及最近几期表现概览；- 点击可跳转到课程详情页，查看课程介绍、统计图表和历史学员评价。4. 授课人排行榜：- 以“授课人”为单位，综合其授课场次的讲师水准评分、课程综合评分和参评人数；- 展示授课人姓名/头衔、平均评分、授课场次、代表课程等信息，突出“大牛”形象；- 点击可跳转至授课人专区，查看其授课简介及各课程的评价统计。5 .会员排行榜：（1）上课排行榜（学习之星）：- 基于会员在指定时间范围内完成上课节数、完成课后评价节数、涉及主题数量等数据计算活跃度；- 展示会员头像、昵称及“学习之星”等标签，仅显示与学习相关的信息，不展示真实姓名；- 引导用户前往课程表或“我的-课程评价”继续预订和完成评价。（2）邀请排行榜（邀请达人）：- 基于邀请成功注册会员人数、被邀请会员购券情况等数据计算邀请贡献；- 展示会员头像、昵称、成功邀请人数、产生购券人数，并与“邀请达人”徽章等级联动；- 引导用户前往“我的-邀请”发出邀请，说明可获得的折扣券奖励。十、会员（“我的”页面）1）个人专区-个人信息卡片，包含昵称、徽章。2）个人专区-个人信息详情：点击详情后展示所有个人信息，包括昵称、姓名、会员id、加入时间、手机号、邀请人昵称（出于信息安全考虑，此处不应该显示邀请人的姓名）、徽章详情。3）个人专区-徽章：邀请达人徽章：根据邀请加入的会员人数计算，1个青铜，3个白银，9个黄金，27个及以上钻石。徽章详情中显示邀请人数和所有被邀请人的昵称（出于信息安全考虑，此处不应该显示邀请人的姓名）。4）授课人专区-讲师资料：预览后台维护的课程介绍、个人背景介绍，修改需要联系管理员。5）授课人专区-课程评价：每个课程的评分概览（本期 4.8，近 3 期平均 00：00 后不再可更新，历史排课信息也不可更新。每一条排课记录右侧增加【取消课程】操作，未开始的课程可以取消，弹出确认框（提示已报名人数、影响）。5）课程管理-问卷配置：可以修改题目和选项。6）评价管理：评论列表（可以修改、隐藏、删除）；关键字过滤设置（脏话）；生成AI总结入口。7）排行榜管理：5种排行榜的生成、修改、发布。8）用户管理-角色列表：3种类型，授课人、会员、游客，这3种基本类型不可变更。9）用户管理-用户列表：存储用户的所有信息，可以新增、删除、修改。10）用户管理-授课人列表：除了在用户列表中存储通用的用户信息以外，此处还需要额外存储个人背景介绍，可以新增、删除、修改。11）用户管理：邀请日志（邀请人、被邀请人、是否完成购券、是否发放激励，不可变更）。12）课券管理：课券列表（一个课券一行记录，可以新增、删除、修改、人工退款按钮）、课券状态变更日志（一次变更一行记录，不可变更）、人工发放课券入口（例如特殊补偿）+日志（不可变更）、购买日志（一次支付一行记录，不可变更）、退款日志（一次调用退款一行记录，不可变更）、发票日志（不可变更）。13）折扣券管理：折扣券发放日志（来源、使用情况，不可变更）、人工发放折扣券入口 。14）消息管理：消息发送日志（用户、类型、发送时间、结果）；课前提醒设置（时间、文字）；课后评价提醒设置（时间、文字）；课券即将过期提醒设置（时间、文字）；邀请成功/获得折扣券提醒设置（时间、文字）。
